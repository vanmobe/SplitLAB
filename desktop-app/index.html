    What SPLITLAB Does

    SPLITLAB separates one song into stems (vocals, drums, bass, other or 2-stem vocals/instrumental) on your own machine.

    Typical Workflow

    1. Start the Engine and run diagnostics.
    2. Choose input file and output folder.
    3. Select stems, quality, model, optional ensemble.
    4. Start split and wait for completion.
    5. Open Player tab, load stems, audition combinations.

    Quality Controls

    • Fast: quickest, lower separation quality.
    • Balanced: best speed/quality tradeoff.
    • High: slowest, often cleanest transients.
    • Preset Best / Vocal Boost: adds more post-processing.

    Model Guidance

    Model

    Primary model used for separation. If unsure, keep Default or choose a known high-quality model from the list.

    Ensemble Model

    Runs a second model and averages outputs. Usually improves quality, but takes roughly 2x compute time.

    When To Use Ensemble

    • Use for final exports and difficult mixes.
    • Skip for quick previews or low-power machines.

    Practical Tip

    Start with Balanced + one model. If artifacts remain, try High + ensemble.

    Demucs Models (DMX / Demucs)

    htdemucs

    Strong all-round model and usually the safest first choice. It gives a good balance between separation quality and runtime. Recommended for most songs when you are starting.

    htdemucs_ft

    Fine-tuned variant that often improves detail and reduces artifacts on complex mixes. It is typically slower than the base model. Best for final exports when quality matters most.

    mdx / mdx_extra

    Can work very well for vocals and difficult bleed cases. Sometimes sounds more aggressive than Demucs-style models, so compare by ear. Great as an ensemble partner when you need cleaner vocal isolation.

    How To Choose

    Use one model first for fast iteration. If results are close, keep the faster one; if both have different strengths, combine them with Ensemble Model. For demanding tracks, try htdemucs_ft + mdx_extra.
